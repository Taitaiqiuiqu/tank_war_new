# 坦克大战游戏素材结构说明

## 一、坦克图片素材结构概述

本项目中的坦克图片素材采用了清晰的分层结构组织，主要为了实现良好的文件分类和资源管理。整个坦克素材位于`tank_images`目录下，分为敌人坦克(`enemy`)和玩家坦克(`player`)两大类别。

## 二、详细目录结构

```
tank_images/
├── enemy/                # 敌人坦克素材
│   ├── e1/               # 敌人坦克类型1
│   │   ├── e1_0/         # 敌人坦克1，等级0
│   │   │   ├── e1_0_0.png  # 敌人坦克1，等级0，精灵帧0
│   │   │   └── e1_0_1.png  # 敌人坦克1，等级0，精灵帧1
│   ├── e2/               # 敌人坦克类型2
│   ├── e3/               # 敌人坦克类型3
│   └── e4/               # 敌人坦克类型4
└── player/               # 玩家坦克素材
    ├── p1/               # 玩家坦克类型1
    │   ├── p1_0/         # 玩家坦克1，等级0
    │   │   ├── p1_0_0.png  # 玩家坦克1，等级0，精灵帧0
    │   │   └── p1_0_1.png  # 玩家坦克1，等级0，精灵帧1
    │   ├── p1_1/         # 玩家坦克1，等级1
    │   ├── p1_2/         # 玩家坦克1，等级2
    │   └── p1_3/         # 玩家坦克1，等级3
    ├── p2/               # 玩家坦克类型2
    ├── p3/               # 玩家坦克类型3
    └── p4/               # 玩家坦克类型4
```

## 三、命名规则说明

坦克图片的命名严格遵循以下格式：

```
[坦克标识]_[等级]_[精灵帧编号].[格式]
```

- **坦克标识**：以`e`开头表示敌人坦克，以`p`开头表示玩家坦克，后跟数字区分不同类型
- **等级**：表示坦克的等级，从0开始递增
- **精灵帧编号**：表示坦克动画的不同帧，从0开始递增

例如：
- `e1_0_0.png`：敌人坦克1，等级0，精灵帧0
- `p2_3_1.png`：玩家坦克2，等级3，精灵帧1

## 四、素材组织原则

1. **分类清晰**：按照敌人/玩家、坦克类型、等级三个维度进行分类，便于快速定位资源
2. **统一管理**：相同类型的资源存放在一起，方便批量加载和管理
3. **动画支持**：通过多个精灵帧实现简单的动画效果，如坦克移动时的履带动画
4. **扩展性好**：结构设计允许轻松添加新的坦克类型或等级

## 五、开发使用建议

1. 在代码中可以通过构建路径字符串来动态加载相应的坦克图片：
   ```
   # 示例：加载玩家坦克1，等级2，精灵帧0的图片
   image_path = f"tank_images/player/p1/p1_2/p1_2_0.png"
   ```

2. 对于动画效果，可以通过循环切换精灵帧来实现：
   ```
   # 示例：在游戏循环中切换精灵帧
   current_frame = 0 if game_time % 100 < 50 else 1
   ```

3. 当坦克升级时，可以动态切换到对应等级的图片资源

## 六、注意事项

- 该结构仅用于文件分类，不包含方向信息。方向相关的处理需要在代码中实现
- 每个等级目录下的精灵帧数量可能不同，建议在代码中动态检测可用的精灵帧数量
- 确保所有图片资源的尺寸统一，以避免在游戏中出现显示异常
