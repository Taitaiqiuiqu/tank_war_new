# 地图文档 (Maps Documentation)

## 可用地图列表

### 1. 经典地图 (classic.json)
**描述**: 经典坦克大战地图布局，包含砖墙、钢墙和草地

**特点**:
- 中央砖墙横排，可以被摧毁开辟通道
- 左右两侧钢墙形成边界保护
- 中上方草地区域提供隐蔽
- 底部中央有基地需要保护

**适合**: 新手玩家，平衡的攻防体验

---

### 2. 迷宫地图 (maze.json)
**描述**: 复杂的迷宫布局，考验策略和走位

**特点**:
- 上下两端有砖墙和钢墙混合结构
- 中部两条砖墙横排形成通道
- 需要绕路才能到达对方区域
- 适合伏击和迂回战术

**适合**: 喜欢策略和走位的玩家

---

### 3. 堡垒地图 (fortress.json)
**描述**: 强调防御的地图，钢墙形成堡垒结构

**特点**:
- 上方左右两个钢墙堡垒
- 中央砖墙堡垒可以被摧毁
- 基地周围有钢墙保护
- 防御性强，适合防守反击

**适合**: 喜欢防御战术的玩家

---

### 4. 过河地图 (river_crossing.json)
**描述**: 河流横贯地图，需要找到合适的通道

**特点**:
- 两条垂直河流将地图分为三个区域
- 河流阻挡坦克但子弹可穿过
- 需要绕路或利用射击角度
- 考验机动性和射击技巧

**适合**: 喜欢机动战的玩家

---

### 5. 草原地图 (grassland.json)
**描述**: 大片草地提供隐蔽，适合伏击战术

**特点**:
- 三个大草地区域可以隐藏坦克
- 草地覆盖坦克但子弹可穿过
- 砖墙和钢墙点缀其间
- 适合隐蔽和伏击战术

**适合**: 喜欢伏击和隐蔽战术的玩家

---

## 地图格式说明

所有地图遵循统一的JSON格式:

```json
{
  "name": "地图显示名称",
  "description": "地图描述",
  "width": 800,
  "height": 600,
  "walls": [
    {"x": 坐标x, "y": 坐标y, "type": 墙体类型}
  ],
  "player_spawns": [
    [P1_x, P1_y],
    [P2_x, P2_y]
  ],
  "enemy_spawns": [
    [E1_x, E1_y],
    [E2_x, E2_y],
    [E3_x, E3_y]
  ]
}
```

### 墙体类型 (Wall Types)
- `1`: 砖块 - 可摧毁，阻挡坦克和子弹
- `2`: 钢块 - 不可摧毁，阻挡坦克和子弹
- `3`: 草地 - 可隐藏坦克，坦克和子弹可穿过
- `4`: 河流 - 坦克不可通过，子弹可穿过
- `5`: 基地 - 玩家基地，可摧毁，需要保护

### 出生点 (Spawn Points)
- **player_spawns**: 玩家出生点，第一个为P1(主机)，第二个为P2(客户端)
- **enemy_spawns**: 敌人出生点，可以有1-3个

---

## 地图设计原则

1. **平衡性**: 确保双方玩家起始位置公平
2. **多样性**: 包含多种墙体类型，提供丰富的战术选择
3. **保护基地**: 基地位于底部中央，需要玩家保护
4. **网格对齐**: 所有墙体坐标为50的倍数，确保对齐
5. **联机支持**: 所有地图支持双人联机模式
