# 音频素材解释

## musics目录结构

```
musics/
├── boom.wav
├── bullet.destroy.wav
├── enemy.move.wav
├── fire.wav
├── player.move.wav
├── start.wav
├── 玩家原地待机音效.mp3
└── 砖块消除.wav
```

## 音频素材使用条件

### boom.wav
- **触发条件**：坦克被击中爆炸时
- **使用方式**：播放一次

### bullet.destroy.wav
- **触发条件**：子弹击中目标（如墙体、坦克）后消失时
- **使用方式**：播放一次

### enemy.move.wav
- **触发条件**：敌方坦克移动时
- **使用方式**：移动时循环播放

### fire.wav
- **触发条件**：坦克发射子弹时
- **使用方式**：每次射击时播放一次

### player.move.wav
- **触发条件**：玩家控制的坦克移动时
- **使用方式**：移动时循环播放

### start.wav
- **触发条件**：游戏开始或关卡开始时
- **使用方式**：播放一次

### 玩家原地待机音效.mp3
- **触发条件**：玩家坦克停止移动但保持激活状态时
- **使用方式**：待机状态下循环播放

### 砖块消除.wav
- **触发条件**：子弹击中并摧毁砖块墙体时
- **使用方式**：每次砖块被摧毁时播放一次

## 音频设计特点

1. **功能明确**：每个文件名都清晰描述了其对应游戏事件
2. **格式多样性**：主要使用wav格式（无损音频，适合游戏音效）和部分mp3格式
3. **分类合理**：覆盖了游戏中几乎所有主要交互事件（移动、射击、爆炸、破坏等）
4. **用户体验导向**：区分了玩家和敌方的音效，增强了游戏的沉浸感和可辨识度
